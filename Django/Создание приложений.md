У нас создаться папка приложения

Вот краткое описание назначения каждого файла и папки в структуре вашего Django-приложения:
- **`nameofapplication/`**  
    Корневая папка приложения (имя может отличаться). Содержит все модули вашего приложения.
- **`migrations/`**  
    Папка для хранения миграций (см. [[Миграции]])
- **`__init__.py`**  
    Пустой файл, который превращает папку в Python-модуль, чтобы можно было импортировать содержимое приложения.
- **`admin.py`**  
    Здесь регистрируются БД (модели) для административной панели Django. Позволяет настраивать, как БД (модели) отображаются и редактируются в `/admin/`.
- **`apps.py`**  
    Определяет класс конфигурации приложения (подкласс `AppConfig`), где можно задать имя приложения и подключить сигналы при старте.
- **`models.py`**  
    Описывает структуры данных вашего приложения в виде классов-моделей (ORM). Django по ним создаёт таблицы в базе.
- **`tests.py`**  
    Содержит тесты (юнит- и функциональные) для вашего приложения. Помогает автоматически проверять корректность кода.
- **`views.py`**  
    Определяет представления (функции или классы), которые обрабатывают HTTP-запросы и возвращают ответы (HTML, JSON и т. п.).

# Регистрация приложений
- Открываем [[settings.py]] и ищем `INSTALLED_APPS`
- Добавляем новый пункт, например 
``` Python
INSTALLED_APPS = [
	'nameofapplication', # Вот наше приложение
    'django.contrib.admin',
    ...
    'django.contrib.staticfiles',
]
```

Открываем [[urls.py]] и добавляем
``` Python
from django.urls import include
```

Также в массив добавим наше приложение
``` Python
urlpatterns = [
	path('nameofapplication', include('nameofapplication.urls'))
	path('admin/', admin.site.urls),
]
```
- 1-й аргумент - это адрес (после слэша), в нашем случае `yoursite.com/nameofapplication`, в случае если хотим изменить главную оставляем пустые кавычки
- 2-й аргумент говорит куда нас переслать, в нашем примере в следующем файле мы откроем `views.function`

Мы заметим что нет файла `nameofapplication/urls.py` => вручную его создаем и добавляем основу
``` Python
from django.urls import path
from . import views

urlpatterns = [
	path('', views.function)
]
```

В [[views.py]] мы уже непосредственно пишем интерфейс страницы, а название функции подставляем вместо `function`
