## 2. Создание проекта Django

``` bash
django-admin startproject NameOfProject
```

У нас создаться папка с проектом, в ней [[manage.py]], также немаловажный [[settings.py]]


## 3.  Запуск проекта

``` bash
cd NameOfProject
```

``` bash
python manage.py runserver
```

По ссылке [http://localhost:8000/](http://localhost:8000/) вы увидете

## 4.  Создание приложений

``` bash
python manage.py startapp nameofapplication
```

У нас создаться папка приложения
Вот краткое описание назначения каждого файла и папки в структуре вашего Django-приложения:
- **`main/`**  
    Корневая папка приложения (имя может отличаться). Содержит все модули вашего приложения.
- **`migrations/`**  
    Папка с автосгенерированными файлами миграций базы данных. Хранит логи изменений моделей (создание/изменение/удаление таблиц и полей).
- **`__init__.py`**  
    Пустой файл, который превращает папку в Python-модуль, чтобы можно было импортировать содержимое приложения.
- **`admin.py`**  
    Здесь регистрируются модели для административной панели Django. Позволяет настраивать, как модели отображаются и редактируются в `/admin/`.
- **`apps.py`**  
    Определяет класс конфигурации приложения (подкласс `AppConfig`), где можно задать имя приложения и подключить сигналы при старте.
- **`models.py`**  
    Описывает структуры данных вашего приложения в виде классов-моделей (ORM). Django по ним создаёт таблицы в базе.
- **`tests.py`**  
    Содержит тесты (юнит- и функциональные) для вашего приложения. Помогает автоматически проверять корректность кода.
- **`views.py`**  
    Определяет представления (функции или классы), которые обрабатывают HTTP-запросы и возвращают ответы (HTML, JSON и т. п.).
