  Обзор

  Backend MMORPG игры представляет собой Django REST Framework приложение, реализующее основные функции для DnD-like
  многопользовательской ролевой игры. Включает в себя системы географии (миры, страны, города), персонажей, а также чатов с
  поддержкой NPC через LLM.

  Технологии

   - Python 3.11+
   - Django 5.2+
   - Django REST Framework
   - Django Channels (для WebSocket)
   - PostgreSQL (основная БД, поддерживается SQLite для разработки)
   - Redis (для Channels)
   - JWT (аутентификация)
   - OpenAI API (для интеграции с DeepSeek LLM)

  Модели данных

  1. Map (Мир)
  Базовая модель игрового мира

  Поля:
   - id - идентификатор (автоматически)
   - name - название мира (уникальное, строка до 255 символов)
   - description - описание мира (текст, опционально)
   - created_at - дата создания (автоматически)
   - updated_at - дата обновления (автоматически)

  2. Country (Страна)
  Модель страны внутри мира

  Поля:
   - id - идентификатор (автоматически)
   - name - название страны (строка до 255 символов)
   - description - описание страны (текст, опционально)
   - map - ссылка на мир (ForeignKey на Map)
   - created_at - дата создания (автоматически)
   - updated_at - дата обновления (автоматически)

  Уникальность: Название страны должно быть уникальным в пределах одного мира

  3. City (Город)
  Модель города с координатами

  Поля:
   - id - идентификатор (автоматически)
   - name - название города (строка до 255 символов)
   - description - описание города (текст, опционально)
   - country - ссылка на страну (ForeignKey на Country)
   - x - координата X (целое число)
   - y - координата Y (целое число)
   - created_at - дата создания (автоматически)
   - updated_at - дата обновления (автоматически)

  Уникальность: Название города должно быть уникальным в пределах одной страны

  4. Character (Персонаж)
  Модель персонажа (игроки и NPC)

  Поля:
   - id - идентификатор (автоматически)
   - name - имя персонажа (строка до 255 символов)
   - character_type - тип персонажа (PC - игрок, NPC - NPC)
   - user - ссылка на пользователя Django (ForeignKey на User, опционально)
   - race - раса персонажа (список вариантов: HUMAN, ELF, DWARF, и др.)
   - char_class - класс персонажа (список вариантов: WARRIOR, MAGE, ROGUE, и др.)
   - level - уровень (положительное целое, по умолчанию 1)
   - health - текущее здоровье (положительное целое, по умолчанию 100)
   - max_health - максимальное здоровье (положительное целое, по умолчанию 100)
   - mana - текущая мана (положительное целое, по умолчанию 50)
   - max_mana - максимальная мана (положительное целое, по умолчанию 50)
   - experience - опыт (положительное целое, по умолчанию 0)
   - description - описание персонажа (текст, опционально)
   - location_x - текущая координата X (целое число)
   - location_y - текущая координата Y (целое число)
   - current_city - текущий город (ForeignKey на City, опционально)
   - current_country - текущая страна (ForeignKey на Country, опционально)
   - current_map - текущий мир (ForeignKey на Map, опционально)
   - created_at - дата создания (автоматически)
   - updated_at - дата обновления (автоматически)

  5. Чаты

  Chat (абстрактная модель)
  Базовая модель для всех типов чатов

  Поля:
   - name - название чата (строка до 255 символов)
   - description - описание чата (текст, опционально)
   - created_at - дата создания (автоматически)
   - updated_at - дата обновления (автоматически)

  WorldChat (Мировой чат)
  Чат для всего мира

  Поля:
   - Все поля от Chat
   - map - ссылка на мир (ForeignKey на Map)

  CountryChat (Чат страны)
  Чат для страны

  Поля:
   - Все поля от Chat
   - country - ссылка на страну (ForeignKey на Country)

  CityChat (Чат города)
  Чат для города

  Поля:
   - Все поля от Chat
   - city - ссылка на город (ForeignKey на City)

  PrivateChat (Приватный чат)
  Чат между персонажами

  Поля:
   - Все поля от Chat
   - participants - участники чата (ManyToMany на Character)

  6. Message (Сообщение)
  Модель сообщения в чате

  Поля:
   - id - идентификатор (автоматически)
   - sender - отправитель (ForeignKey на Character)
   - content - содержание сообщения (текст)
   - message_type - тип сообщения (TEXT, SYSTEM, NOTIFICATION, по умолчанию TEXT)
   - timestamp - время отправки (автоматически)
   - world_chat - ссылка на мировой чат (ForeignKey на WorldChat, опционально)
   - country_chat - ссылка на чат страны (ForeignKey на CountryChat, опционально)
   - city_chat - ссылка на чат города (ForeignKey на CityChat, опционально)
   - private_chat - ссылка на приватный чат (ForeignKey на PrivateChat, опционально)

  Ограничения: Сообщение должно быть отправлено ровно в один тип чата (только одно из полей chat-а может быть заполнено)

  Аутентификация

  JWT Token
  Все API эндпоинты (кроме аутентификации) требуют JWT токен в заголовке:

   1 Authorization: Bearer <access_token>

  Token получения
   1. Получение токенов:

   1    POST /api/auth/token/
   2    Content-Type: application/json
   3    
   4    {
   5      "username": "user",
   6      "password": "password"
   7    }

     Ответ:

   1    {
   2      "refresh": "refresh_token",
   3      "access": "access_token",
   4      "characters": [
   5        // данные персонажей пользователя
   6      ]
   7    }

   2. Регистрация:

   1    POST /api/auth/register/
   2    Content-Type: application/json
   3    
   4    {
   5      "username": "new_user",
   6      "email": "email@example.com",
   7      "password": "password"
   8    }

     Ответ:

   1    {
   2      "refresh": "refresh_token",
   3      "access": "access_token",
   4      "user": {
   5        "id": 1,
   6        "username": "new_user",
   7        "email": "email@example.com"
   8      }
   9    }
