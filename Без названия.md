# Оценка глубины по одиночному изображению (Monocular Depth Estimation)

**Оценка глубины** – задача восстановления расстояния от камеры до объектов на изображении на основе единственного RGB-кадра. Это важный шаг при реконструкции сцены и используется в мобильной робототехнике, автопилотах, дополненной реальности и др. Классически модель строится на сверточных нейронных сетях (CNN): из входного цветного изображения предсказывается карта глубины, где каждому пикселю соответствует число-расстояние ([Monocular depth estimation](https://keras.io/examples/vision/depth_estimation/#:~:text=Depth%20estimation%20is%20a%20crucial,convnet%20and%20simple%20loss%20functions)). Ниже приведено описание возможной реализации на TensorFlow/Keras, включая выбор датасетов, предобработку, архитектуру, обучение, инференс и экспорт модели.

## Открытые датасеты для обучения

Для обучения необходимы пары “RGB – карта глубины”. Существуют несколько общедоступных датасетов:

- **NYU Depth V2** – набор **интерьерных** сцен, снятых камерой Kinect. Содержит ~1449 плотных пар RGB‑изображений и карт глубины с заполненными пропусками ([NYU Depth V2 « Nathan Silberman](https://cs.nyu.edu/~fergus/datasets/nyu_depth_v2.html#:~:text=The%20NYU,It%20features)). Это классический датасет для обучения моделей внутри зданий.
    
- **KITTI** – набор **уличных** (дорожных) сцен, собранный с автомобиля: два стереокамеры и лидар Velodyne. Доступны разреженные карты глубины (LiDAR-проекции) для 86k тренировочных изображений ([KITTI-Depth Dataset | Papers With Code](https://paperswithcode.com/dataset/kitti-depth#:~:text=The%20%2A%2AKITTI,access%20to%20the%20ground%20truth)). Точные текстовые описания KITTI подчёркивают, что «ground truth предоставляется лазерным сканером Velodyne» ([The KITTI Vision Benchmark Suite](https://www.cvlibs.net/datasets/kitti/#:~:text=odometry%2C%203D%20object%20detection%20and,size)).
    
- **TUM RGB-D** – последовательности **интерьерных** сцен от Kinect, с акселерометром и точной привязкой к траектории. Содержит 30 ГБ данных цветных и depth-изображений (модель Kinect v1) с полным 30 FPS ([Computer Vision Group - Datasets - RGB-D SLAM Dataset and Benchmark](https://cvg.cit.tum.de/data/datasets/rgbd-dataset#:~:text=visual%20SLAM%20systems,640x480)). Подходит для исследований SLAM и может быть использован в качестве дополнительного набора.
    
- **Cityscapes** и **Make3D** – другие примеры: Cityscapes содержит уличные сцены с парой камер (можно получить диспаритет), Make3D – небольшой набор из 400 улиц с глубинами. Также существуют синтетические датасеты и другие источники (или пользовательские записи, например с камеры Intel RealSense).
    

Для практики с Keras часто пользуются более компактными наборами или TFDS: например, примеры Keras по глубине берут датасет DIODE, но рекомендовали использовать NYU или KITTI ([Monocular depth estimation](https://keras.io/examples/vision/depth_estimation/#:~:text=We%20will%20be%20using%20the,v2%20and%20KITTI)). Инициировать загрузку NYU через TensorFlow Datasets можно так:

```python
import tensorflow_datasets as tfds
ds = tfds.load('nyu_depth_v2', split=['train','validation'], as_supervised=True)
```

При этом NYU-Depth V2 загружается из официального источника ([nyu_depth_v2  |  TensorFlow Datasets](https://www.tensorflow.org/datasets/catalog/nyu_depth_v2#:~:text=)) (около 75 ГБ). Для ускоренного прототипирования можно скачать часть данных или использовать только некоторые сцены.

## Подготовка данных (предобработка)

Изображения и карты глубины обычно приводят к единому размеру и диапазону. Общие шаги предобработки:

- **Изменение размера:** сети часто обучаются на фиксированном разрешении (например, 256×256 или 224×224). При использовании `tf.data` или `keras.preprocessing` применяем `tf.image.resize()`. При этом глубины (обычно в метрах) тоже масштабируем до той же размерности.
    
- **Нормализация:** RGB-изображение переводят в тип float и масштабируют в [0,1] (деление на 255). Карта глубины чаще оставляется как float (метры) или тоже нормируется (например, делением на максимальное ожидаемое значение), чтобы ускорить сходимость.
    
- **Маскирование/отбрасывание:** у KITTI многие пиксели глубины отсутствуют (лидар дает ~5% пикселей ([KITTI-Depth Dataset | Papers With Code](https://paperswithcode.com/dataset/kitti-depth#:~:text=The%20%2A%2AKITTI,access%20to%20the%20ground%20truth))). Обычно их маскируют и не учитывают в функции потерь (или интерполируют).
    
- **Аугментации:** для генерализации можно применять случайные повороты, сдвиги или изменение яркости к RGB-кадрам и соответствующим глубинам.
    

_Пример кода предобработки:_

```python
import tensorflow as tf

def preprocess(image, depth, target_size=(256, 256)):
    # Приводим к float32
    image = tf.cast(image, tf.float32) / 255.0
    depth = tf.cast(depth, tf.float32)  # глубина в метрах
    # Изменяем размер до target_size
    image = tf.image.resize(image, target_size)
    depth = tf.image.resize(depth[..., tf.newaxis], target_size)[..., 0]
    return image, depth

# Пример создания tf.data Dataset (RGBD пары из TFDS):
train_ds = ds[0].map(lambda img, d: preprocess(img, d))
train_ds = train_ds.batch(8).prefetch(1)
```

В Keras-примере для DIODE используют собственный генератор `DataGenerator`, но принципы те же ([Monocular depth estimation](https://keras.io/examples/vision/depth_estimation/#:~:text=HEIGHT%20%3D%20256%20WIDTH%20%3D,30%20BATCH_SIZE%20%3D%2032)).

## Архитектура модели

Обычно используется **конволюционная сжатая-расжатая сеть** (encoder-decoder/U-Net): сверточная часть («энкодер») извлекает признаки, а «декодер» транспонированными свертками или upsamping реконс­труирует карту глубины. Можно применять готовые предобученные энкодеры (VGG, ResNet, EfficientNet) и учить только декодер. Например, простая U-Net:

```python
from tensorflow import keras
from tensorflow.keras import layers

input_img = keras.Input(shape=(256, 256, 3))
# Энкодер (сокращаем простыми свертками+пулом):
x = layers.Conv2D(64, 3, activation='relu', padding='same')(input_img)
x = layers.MaxPooling2D()(x)
x = layers.Conv2D(128, 3, activation='relu', padding='same')(x)
x = layers.MaxPooling2D()(x)
x = layers.Conv2D(256, 3, activation='relu', padding='same')(x)
x = layers.MaxPooling2D()(x)
# Декодер (расшираем обратно):
x = layers.Conv2DTranspose(256, 3, strides=2, padding='same', activation='relu')(x)
x = layers.Conv2DTranspose(128, 3, strides=2, padding='same', activation='relu')(x)
x = layers.Conv2DTranspose(64, 3, strides=2, padding='same', activation='relu')(x)
# Финальный слой: одна карта глубины (линейная активация)
output = layers.Conv2D(1, 3, activation='linear', padding='same')(x)

model = keras.Model(inputs=input_img, outputs=output)
model.summary()
```

Такая сеть выдаёт карту глубины того же размера, что и входное RGB. В Keras-примере использовался похожий подход с UpSampling2D и свёртками (код блока декодера показан ниже) ([Monocular depth estimation](https://keras.io/examples/vision/depth_estimation/#:~:text=self,Concatenate)). Можно усиливать архитектуру: добавлять скипы из энкодера, использовать BatchNorm, LeakyReLU, более глубокие сети и т. д. Популярны U-Net, FCRN, ResNet-декодеры и современные модели типа **DenseDepth** или **AdaBins**, но для примера достаточно простого конволюционного автокодировщика.

## Обучение модели

Модель обучаем подгонкой предсказанной карты глубины под эталон (насколько правильно спрогнозирован каждый пиксель). Типичные параметры компиляции:

```python
model.compile(
    optimizer=keras.optimizers.Adam(1e-4),
    loss='mse',  # или 'mae'
    metrics=['mae']
)
```

Можно также использовать комбинированные функции потерь. Например, в Keras-примере одновременно оптимизировали метрику **SSIM** (похожесть структур), L1-потерю и сглаживание карты глубины ([Monocular depth estimation](https://keras.io/examples/vision/depth_estimation/#:~:text=We%20will%20optimize%203%20losses,Depth%20smoothness%20loss)). Они отмечают, что SSIM даёт наибольший вклад в качество ([Monocular depth estimation](https://keras.io/examples/vision/depth_estimation/#:~:text=We%20will%20optimize%203%20losses,Depth%20smoothness%20loss)). Для упрощения можно ограничиться среднеквадратичной ошибкой (MSE).

_Обучение:_

```python
history = model.fit(
    train_ds, 
    validation_data=val_ds, 
    epochs=20,
    callbacks=[keras.callbacks.ModelCheckpoint("best_depth_model.h5", save_best_only=True)]
)
```

Здесь `train_ds` и `val_ds` – предварительно подготовленные батчированные датасеты (например, из TFDS или кастомных путей к файлам). Количество эпох и размер батча зависит от объёма данных и ресурсов.

## Инференс (предсказание глубины)

После обучения получим модель, которая на входе RGB возвращает depth-map. Пример кода инференса на одном изображении:

```python
import numpy as np
import matplotlib.pyplot as plt

# Предобработаем одно тестовое изображение (например, загружаем 256x256)
test_image = tf.io.read_file("test.jpg")
test_image = tf.image.decode_image(test_image, channels=3)
test_image = tf.image.resize(test_image, (256,256))
test_image = tf.cast(test_image, tf.float32) / 255.0
test_image = tf.expand_dims(test_image, 0)  # батч из одного

# Предсказание карты глубины
pred_depth = model.predict(test_image)[0, ..., 0]

# Визуализация
plt.imshow(pred_depth, cmap='plasma')
plt.colorbar(label='Глубина (условные ед.)')
plt.title('Предсказанная карта глубины')
plt.show()
```

В реальном времени можно организовать цикл: считывать кадр с камеры (через OpenCV, ROS, TensorFlow InputPipeline и т.д.), предобрабатывать, вызывать `model.predict` и передавать результат дальше (например, в навигационный модуль). Предсказание одной карты обычно занимает доли секунды на GPU.

## Экспорт и интеграция модели в робототехнику

Для использования модели на роботе или в приложении полезно оптимизировать и экспортировать её:

- **Сохранение TensorFlow-модели:** можно сохранить в формате _SavedModel_ или HDF5: `model.save("depth_model")`. Затем эта модель загружается в любом TF-совместимом окружении.
    
- **TensorFlow Lite (TFLite):** конвертация в TFLite позволяет запускать модель на мобильных и встроенных устройствах с аппаратным ускорением. Пример конвертации Keras-модели:
    
    ```python
    import tensorflow as tf
    converter = tf.lite.TFLiteConverter.from_keras_model(model)
    # Оптимизации, квантование (опционально)
    converter.optimizations = [tf.lite.Optimize.DEFAULT] 
    tflite_model = converter.convert()
    open('depth_model.tflite', 'wb').write(tflite_model)
    ```
    
    Как описано в официальной документации, после этого файл `.tflite` можно загрузить в C++/Java на мобильных платформах или на встраиваемых устройствах ([Convert TensorFlow models  |  Google AI Edge  |  Google AI for Developers](https://ai.google.dev/edge/litert/models/convert_tf#:~:text=converter%20%3D%20tf.lite.TFLiteConverter.from_saved_model%28saved_model_dir%29%20,convert)).
    
- **TensorRT или Edge TPU:** для NVIDIA Jetson и похожих платформ можно использовать TensorRT (TF-TRT) для ускорения. Google Coral Edge TPU требует дополнительного квантования до int8, но даёт очень быструю инференс-производительность. Все эти опции позволяют интегрировать глубинную модель в автономный робот (например, использовать глубину для избегания препятствий или построения карты).
    
- **ROS:** если система построена на Robot Operating System, можно написать узел на Python/C++, который читает кадры из ROS-топика, выполняет инференс модели и публикует карту глубины или извлекает из неё объекты. Например, с помощью `ros_numpy` и `rospy` можно преобразовывать сообщения `sensor_msgs/Image` в NumPy-массивы, и обратно. Таким образом модель оценщика глубины становится частью пайплайна робота.
    

Оптимизация и экспорт модели подробно описаны в официальной документации TensorFlow Lite ([Convert TensorFlow models  |  Google AI Edge  |  Google AI for Developers](https://ai.google.dev/edge/litert/models/convert_tf#:~:text=converter%20%3D%20tf.lite.TFLiteConverter.from_saved_model%28saved_model_dir%29%20,convert)). Используя эти инструменты, можно запускать получение глубины в реальном времени на встраиваемых компьютерах или мобильных процессорах.
