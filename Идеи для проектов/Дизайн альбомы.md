---
### *Проблема*

Google Photos и Apple Photos создают авто-альбомы («воспоминания», «этот день в прошлые годы»), но дизайн **шаблонный и безличный** — одни и те же слайд-шоу у всех. Нет учёта настроения, эстетики, контекста. Спрос на красивую упаковку воспоминаний огромный: рынок фото-книг уже ~$1B+ TAM (Artifact, Chatbooks, Printful), люди готовы платить за журналы и книги. Проблема не в «где хранить фото», а в **«как красиво пересказать момент»**.

---  
### *Решение*

**Standalone web-app** (не плагин — не зависеть от платформы) с опциональными интеграциями Google Photos / iCloud / локальный импорт. Делает **персонализированные альбомы с дизайном ручной работы**:

1. **Дедупликация.** Open-source perceptual hash (imagehash / pHash + DBSCAN) находит одинаковые / почти одинаковые фото. На каждый кластер выбирается «лучшее» (резкость, лица, экспозиция через CLIP-эмбеддинги).

2. **Vision-LLM теги.** Лёгкая мультимодальная модель (Gemini 2.5 Flash или Qwen2-VL) вешает на каждое фото вектор вайба: палитра, яркость, сцена, время суток, состав людей, эмоция. На старте — **4–6 формализованных vibe-категорий** (chill / energetic / intimate / epic / cozy / wild). Не «магия», а классификация.

3. **LLM-аналитик шаблона.** Тяжёлая модель (Claude Sonnet / Gemini Pro) смотрит на агрегированные теги и предлагает **шаблон из библиотеки** (5–10 hand-crafted на категорию, v1). Идёт в чат с пользователем с уточняющими вопросами: «Это путешествие? Праздник? Кто главные герои? Какой тон подписей — с юмором или поэтичный?». **Один round-trip, не больше** (cost-control).

4. **Агент-вёрстка.** Агент выбирает конкретные фото под слоты шаблона, расставляет на холсте с учётом композиции, генерит подписи под вайб. **Cap на итерации: максимум 5 vision-проверок** на альбом, чтобы cost не ушёл в небо.

5. **UX-правки.** Drag-and-drop, регенерация отдельных подписей, замена фото. **Кешируем выбор фото→шаблон** — перегенерим только подписи при правке текста.

**Гибрид шаблонов:** базовые макеты — от дизайнеров вручную (качество и маркетинг-история), вариации и tone-of-voice — AI. На v2 — **маркетплейс шаблонов** от дизайнеров (premium-покупки).

**Экспорт:**
- **Цифровой:** PDF, видео-slideshow, shareable-ссылка (с explicit privacy opt-in per album)
- **Печатный:** фото-книга через **Prodigi / Gelato API** (zero capex, маржа 20–40%)
- **Музыка** — в v2 (Suno/Mubert API, нужен license control)

### *Заработок*  

| Источник                                                                                   | Цена           | Маржа                            | Приоритет |
| ------------------------------------------------------------------------------------------ | -------------- | -------------------------------- | --------- |
| **Freemium** — 3 альбома / мес бесплатно, Premium $5/мес безлимит + AI-тексты + HD-экспорт | $5/мес         | **~89%** (при cost $0.18/альбом) | v1        |
| **Печать** — партнёрство с Prodigi / Gelato, комиссия 20–40% с заказа фото-книги           | $20–60/заказ   | 20–40%                           | v1        |
| **B2B** — свадебные фотографы, event-агентства, отели (альбомы для гостей как услуга)      | $50–200/проект | ~70%                             | v2        |
| **Маркетплейс шаблонов** — премиум-шаблоны от дизайнеров                                   | $2–10/шаблон   | 70% (комиссия)                   | v2        |
  
**Unit-экономика на 1 premium-юзера:** платит $5, создаёт ~10 альбомов/мес = $1.8 cost → **$3.2 маржа/юзера/мес **(64% gross margin). Окупаемость AI-cost при конверсии freemium→premium** >3%**.

#### _Go-to-Market (roadmap)_

**Этап 0 — Custdev (1 неделя, $0).** 10 интервью со скриншотами Google Photos auto-albums + 2–3 hand-crafted шаблона. Вопрос: «Купил бы за $5/мес?». Без этого шага — не пилить.

**Этап 1 — Lean MVP, стратегия A (4–6 недель, $2–5k).** Standalone web-app (Next.js). Импорт фото → дедуп → 3 дизайнерских шаблона (ручная вёрстка, без AI) → PDF + печать через Prodigi. Цель: проверить, **покупают ли люди красивую фото-книгу** до того, как тратимся на AI.

**Этап 2 — AI-pipeline, стратегия B (6–10 недель, $5–15k).** Подключаем Gemini Flash для тегов + Sonnet для аналитика + агент-вёрстку. Качество растёт, cost растёт, но **уже понятно, что спрос есть**.

**Этап 3 — Scale.** Self-hosted vision-модели (Qwen2-VL на GPU) для снижения cost. Маркетплейс шаблонов. B2B-тариф. Видео-slideshow.

**Этап 4 — V2.** Музыка (Suno API), Pro-тариф $20/мес для фотографов, white-label для B2B-отелей.

#### _Риски и митигация_

| Риск                                              | Вероятность | Импакт                   | Митигация                                                              |
| ------------------------------------------------- | ----------- | ------------------------ | ---------------------------------------------------------------------- |
| **Зависимость от Google Photos API**              | Средняя     | Высокий                  | Standalone-first, Google Photos — лишь один из каналов импорта         |
| **AI-cost съедает маржу на freemium**             | Средняя     | Высокий                  | Hard cap 30 фото / альбом, low-detail vision для тегов, кеш embeddings |
| **Re-runs юзера взрывают cost**                   | Высокая     | Средний                  | Cap 5 итераций на правки, soft-prompt «сохранить и продолжить»         |
| **Дубликаты и похожие сервисы** (Canva, Artifact) | Высокая     | Высокий                  | Дифференциатор — vibe-matching + B2B-канал, не UI                      |
| **Утечка личных фото через shareable-ссылки**     | Низкая      | Критический (reputation) | Explicit per-album privacy, opt-in share, watermark, 30-day expiry     |
| **Качество AI-вёрстки** «пластиковое»             | Средняя     | Средний                  | Гибрид: дизайнерские базовые макеты, AI только варьирует тон           |

#### _MVP-стек (стратегия A → B)_

- **Frontend:** Next.js 15 + Tailwind + shadcn/ui
- **Backend:** Python FastAPI + Postgres + Redis (кеш embeddings и подписей)
-  **Дедуп:** Python imagehash + DBSCAN (scikit-learn)
- **Vision (этап 2):** Gemini 2.5 Flash API → fallback self-hosted Qwen2-VL-7B
- **LLM-аналитик (этап 2):** Claude Sonnet 4.5
- **Вёрстка:** PptxGenJS / pdf-lib для PDF, ffmpeg для slideshow
- **Печать:** Prodigi API (white-label, без склада)
- **Хранение:** S3 (фото 30 дней), Cloudflare R2 для готовых PDF (egress дёшево)
- **Платежи:** Stripe + Lemon Squeezy (для RU/СНГ)
