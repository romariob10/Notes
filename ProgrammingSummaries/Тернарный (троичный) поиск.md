**Тернарный поиск**  — метод поиска минимума или максимума функции на отрезке, которая либо сначала строго возрастает, затем строго убывает, либо наоборот.


|                      | Бинарный   | Тернарный                                                                               |
| -------------------- | ---------- | --------------------------------------------------------------------------------------- |
| требование к функции | Монотонная | существует $x_{min}$ до которой функция убывающая, а после возростающая или же наоборот |

#### Рассмотрим этот алгоритм на примере поиска минимума (поиск максимума аналогичен).

Пусть функция $f(x)$ на отрезке $[l,r]$ имеет минимум, и мы хотим найти точку $x_{min}$ в которой он достигается.

Посчитаем значения функции в точках $a=l+\frac{r−l}{3}$ и $b=l+2\frac{r−l}{3}$.

Так как в точке $x_{min}$ минимум, то на отрезке $[l,x_{min}]$ функция убывает, а на $[x_{min},r]$ — возрастает, то есть

$∀x',x''∈[l,r]$:
$l<x'<x''<x_{min}⟹f(l)>f(x')>f(x'')>f(x_{min})$
$x_{min}<x'<x''<r⟹f(x_{min})<f(x')<f(x'')<f(r)$

Значит если $f(a)<f(b)$, то $x_{min}\in[l,b]$, аналогично из $f(a)>f(b)$ следует $x_{min}\in[a,r]$.

Тогда нам нужно изменить границы поиска и искать дальше, пока не будет достигнута необходимая точность, то есть $r−l<eps$.

Из рассуждений и рисунка может возникнуть идея взять, например, отрезок $[l,a]$ вместо отрезка $[l,b]$. Но этого делать нельзя, потому что мы не умеем различать случаи, когда $f(a)<f(b)$ и aa слева или справа от минимума.

Можно заметить, что если мы всегда будем брать отрезок $[l,b]$ при $f(a)<f(b)$ или $[a,r]$ при $f(a)>f(b)$ , то минимум функции всегда будет в нашем отрезке. Если $f(a)=f(b)$, то можно взять любой отрезок.

```C++
double ternarySearch(double f(), double L, double R, double eps) {
  double a, b;
  while (R - L > eps) {
    a = (L * 2 + R) / 3;
    b = (L + R * 2) / 3;
    if (f(a) < f(b)) R = b else L = a
  }
  return (L + R) / 2
}
```
